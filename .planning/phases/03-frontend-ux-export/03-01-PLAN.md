Notes on the exact attributes (all REQUIRED, do not drop):
- `novalidate` on the form: we drive inline validation from JS (03-02) with controlled message wording, so we suppress the browser's native bubble UI. `minlength="3"` and `required` stay on the input as defense-in-depth / semantic hints only (see 03-RESEARCH.md INPUT-02).
- `id="input-error"` starts with `hidden` and `role="alert"` — this is the inline validation feedback slot for the <3-char case (INPUT-02). app.ts toggles `hidden` and sets its `textContent`.
- `id="generate-btn"` includes `disabled:opacity-50 disabled:cursor-not-allowed` so the in-flight disabled state (INPUT-03, wired in 03-02) is visually obvious.

```html
<div id="results" class="mt-6 hidden flex flex-col gap-4"></div>
```
Region rules:
- Each region has BOTH a layout class (e.g. `flex`) AND `hidden`. `hidden` (Tailwind = `display:none`) wins by specificity when present; app.ts reveals a region by removing `hidden`. Keep both classes in the markup so app.ts only ever adds/removes the single `hidden` token.
- `#progress` contains a spinner span AND a `#progress-label` span — the label text ("Searching pages...", etc.) is set by app.ts (UI-02). This satisfies "step-labeled progress, not a bare spinner": the label is the required part, the spinner is decorative.
- `#progress` uses `role="status"` + `aria-live="polite"`; `#error` and `#input-error` use `role="alert"`.
- `#results` uses `flex flex-col gap-4` → single column at every width (UI-04), no `sm:`/`md:` grid override (single-column is the only hard requirement per 03-RESEARCH.md).

Do NOT add a `client:load` (or any `client:*`) directive — those are for UI-framework components only and are a no-op here (03-RESEARCH.md Pitfall 1). Do NOT inline the JS in this file; app.ts is a separate module owned by 03-02.
