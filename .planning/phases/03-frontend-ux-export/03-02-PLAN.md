**Simulated progress (UI-02)** — client-side timed labels, NOT real streaming (backend does not stream — 03-RESEARCH.md Progress UX):
- Keep an array of timeout ids: `let progressTimers: number[] = [];`
- `startProgress()`: `hide(#error); hide(#results); show(#progress)`. Set `#progress-label` textContent to the FIRST label immediately (`"Searching pages..."`), then schedule the rest with `setTimeout`, storing every id in `progressTimers`. EXACT schedule:

  - `4000ms` → `"Analyzing content..."`
  - `9000ms` → `"Generating ideas..."`
  - `16000ms` → `"Almost done..."`

- `stopProgress()`: `progressTimers.forEach(clearTimeout); progressTimers = []; hide(#progress);` — MUST be called the instant the fetch settles so a stale label never shows after results/error render (03-RESEARCH.md Pitfall 3).

**Error taxonomy (UI-03)** — `mapError(code, backendMessage)` returns the user-facing string. EXACT mapping (do not paraphrase differently):
```
VALIDATION      → use the backend message verbatim (already clear, e.g. "Keyword must be at least 3 characters"); fall back to "Please enter a valid keyword." if message is empty
NO_RESULTS      → "No results found for that keyword — try a broader or different term."
RATE_LIMITED    → "This tool hit a free-tier rate limit — please try again in a bit."
UPSTREAM_ERROR  → "Something went wrong generating ideas — please try again."
INTERNAL        → "Something unexpected went wrong — please try again."
(unknown / unrecognized code) → "Something unexpected went wrong — please try again."   // future-proof default, never blank
```

**Submit handler (INPUT-01 + INPUT-03):**
```
form.addEventListener('submit', async (e) => {
  e.preventDefault();                       // stop full-page nav; Enter and click both route here
  const keyword = keywordInput.value.trim();
  if (keyword.length < 3) {                 // INPUT-02: mirrors backend z.string().trim().min(3)
    showInlineError('Keyword must be at least 3 characters');
    return;                                 // NO network request
  }
  clearInlineError();
  generateBtn.disabled = true;              // INPUT-03: set synchronously BEFORE await
  startProgress();                          // UI-02
  try {
    const res = await fetch('/api/generate', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },   // REQUIRED — Pitfall 5
      body: JSON.stringify({ keyword }),
    });
    const data = await res.json();          // body is { ideas } or { error:{code,message} } — always JSON
    if (!res.ok) { showError(mapError(data.error?.code, data.error?.message)); return; }
    renderCards(data.ideas as VideoIdea[]);
  } catch {
    showError("Can't reach the server — check your connection and try again.");   // fetch threw, no response
  } finally {
    stopProgress();                         // clears timers even on the success path
    generateBtn.disabled = false;           // re-enable on BOTH success and failure
  }
});
```
